Goals

The purpose is to permit the reproduction of statistical tests on astrology with more complete and reliable data.
With the availability of birth certificates online, this is possible. Checking thousands of birth dates is a big task, but it's really easier than at Gauquelin epoch.
One goal is to reproduce the analysis of Suitbert Ertel described in the book "The mars effect".
Main steps are described in the home page of g5 documentation:
  1. Gather the historical data, check them, correct them, and merge them in a single database.
  2. Connect the historical data to standard datasets like wikidata and complete birth times to build larger groups.
  3. Build new eminence indicators and sort famous persons by eminence rank.

Need for open data

Open data is a necessity to produce data usable for science.
  • Open data
    Scientific works need to be verifiable and reproductible. Any user should have the possibility to download the data on his or her machine, and do what they want with them.
  • Free software (open source)
    Open data could be generated by proprietary (closed) software, because the important thing is data.
    But the choice of opengauquelin.org is to use only free software, which permits easier verification and debug.

What is good data ?

The definition of "good data" used in this program is :
  1. Expressed in open and standard formats.
    • Encoded in UTF-8
      The program does not handle other character encodings. Raw data are converted to UTF-8, generated data are UTF-8.
    • Use only open formats
      The program uses txt, csv, json and yaml.
    • Express dates in a standard format
      The program uses ISO 8601 (YYYY-MM-DD HH:MM).
      This choice is not completely satisfactory because ISO 8601 is not fully available for free.
    • Express countries in a standard format
      The program uses ISO 3166 2-letter codes.
    • Relate places to a standard
      The program uses geonames.org.
    • Use a standard list of occupation codes
      among all data. A convenient standard list was not found. The program uses a home-made list of codes
      Most of g5 are occupations are related to Wikidata, but Wikidata was not used as reference list because some codes are missing (eg "Conductor of military band").
  2. Precise and identifiable without ambiguity.
    • Different forms of a person's name are sometimes needed :
      • Exact spelling of family and given names used in birth certificates.
      • Name commonly used to designate the person.
      • Spouse name for women.
      • Nickname or stage name.
    • Specify the gender of a person.
    • Birth date and times should be expressed exactly as indicated in civil registries.
    The structure of an ogdb person is described by github.com/tig12/gauquelin5/blob/master/src/model/Person.yml.
  3. Verified and verifiable.
    Original document (birth certificate) should be available to the user.
    Under development, not finished
  4. Connected to other data sets.
    Persons should be related to at least one standard id.
    Currently not done by the program
    but see preparatory study on tig12.github.io/gauquelin5/wikidata.html.

Trust level

Trust level is an indicator to measure data reliability.
It defines 4 levels of reliability based on original documents.
A trust level is granted to a data if the original document is available and verifiable by anyone.
  • 1 - Hospital Certificate (HC)
  • 2 - Birth Certificate (BC)
  • 3 - Birth Record (BR)
    (= copy of the BC by an officer)
  • 4 - To Check
    Data possibly containing errors: data coming from Gauquelin, Müller, skeptic groups, wikipedia and other web pages.
    Need to be matched against a BC.
Most births that occured in the 19th and early 20th century do not have Hospital Certificate.
So the best we can hope for OGDB is to reach level 2.
Currently, almost all data of OGDB are level 4.

Remark : Françoise and Michel Gauquelin, and Arno Müller were were apparently taking a lot of care to avoid errors. But in practice, they didn't have access to Birth Certificates: they were writing to the town halls ; an officer was copying the information contained in the BC and sending them back a letter. They were only recieving Birth Records, level 3.

Gauquelin data are considered level 4 because of error sources:
  • The transcription from BC to BR.
  • Gauquelin or Müller error when integrating the BR in their files.
  • Error when original paper files where transcribed in an electronic form.
  • Bugs in g5 program
