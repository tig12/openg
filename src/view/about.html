Goals

The goal is to produce data usable for science. This implies several things:
  • Open data
    Scientific works need to be verifiable and reproductible. Any user should have the possibility to download the data on his or her machine, and do what they want with them.
  • Free software (open source)
    Open data could be generated by proprietary (closed) software, because the important thing is data.
    But the choice of opengauquelin.org was to use only free software. It makes verification and debug easier and produces better quality data.

Data reliability

The data contained in OGDB integrate many corrections, so data are more reliable than the sources it uses.
But one must be aware that OGDB cannot be considered as reliable. It probably still contains many errors and the only way to be sure of the data is to check each person one by one, comparing the values stored in the database with the values written in civil registries.

Trust level

To measure data reliability, an indicator was built and is used in this database.
It defines 5 levels of reliability:
  • 1 - Hospital Certificate (HC)
    Original document available and verifiable by anyone.
  • 2 - Birth Certificate (BC)
    Original document available and verifiable by anyone.
  • 3 - Birth Record (BR)
    (= copy of the BC by an officer - may contain mistakes)
    Original document available and verifiable by anyone.
  • 4 - to check
    Data coming from sources à priori serious (Gauquelin, Müller) but in practice containing errors.
    Need to be matched against BC.
  • 5 - the rest
    Data without birth time or grabbed from the web (wikipedia, web sites).
Most births that occured in the 19th and early 20th century do not have a HC. So the best we can hope for OGDB is to reach level 2.

Currently, almots all data of OGDB are level 4.

Important remark : Françoise and Michel Gauquelin, and Arno Müller were apparently very serious and took a lot of care to avoid errors. But in practice, they never had access to BCs: they were writing to the town halls ; an officer was copying the information contained in the BC and sending them back a letter. They were only recieving BRs.

So here is a first source of error, the transcription from BC to BR.
Other sources of errors exist:
  • Gauquelin or Müller error when integrating the BR in their files.
  • Error when original paper files where put in an electronic form.
  • Bugs in g5 program should be added to this list...
