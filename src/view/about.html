Goals

Goals of OGDB are described on home page of g5 documentation:
  1. Gather the historical data, check them, correct them, and merge them in a single database.
  2. Connect the historical data to standard datasets like wikidata and complete birth times to build larger groups.
  3. Build new eminence indicators and sort famous persons by eminence rank.

Need for open data

Open data is a necessity to produce data usable for science.
  • Open data
    Scientific works need to be verifiable and reproductible. Any user should have the possibility to download the data on his or her machine, and do what they want with them.
  • Free software (open source)
    Open data could be generated by proprietary (closed) software, because the important thing is data.
    But the choice of opengauquelin.org was to use only free software. It makes verification and debug easier and produces better quality data.
One of the purposes is to restore the groups of sportsmen, in order to have a reproduction of Ertel's work about the mars effect, described in book The Tenacious Mars effect.

What is good data ?

The definition of "good data" used in this program is :
  1. Expressed in open and standard formats.
    • Encoded in UTF-8. The program does not handle other character encodings. Raw data encoded in other encodings still in use in some systems must be converted before being processed. All generated data use UTF-8.
    • Use only open formats. The program uses csv, txt, json and yaml.
    • Express dates in a standard format. The program uses ISO 8601. This choice is not completely satisfactory because ISO 8601 is not fully available for free.
    • Express countries in a standard format. The program uses ISO 3166 2-letter codes.
    • Relate places to a standard. The program uses geonames.org.
    • Use a standard list of occupation codes among all data. A convenient standard list was not found. The program uses a home-made list of codes (which is not satisfactory, but at least operational and consistent among generated files).
      Most of g5 are occupations are related to Wikidata, but Wikidata was not used because come codes are missing (eg "conductor of military band").
  2. Precise and identifiable without ambiguity.
    • Different forms of a person's name are sometimes needed :
      • Exact spelling of family and given names used in birth certificates.
      • Name commonly used to designate the person.
      • Spouse name for women.
      • Nickname or stage name.
    • Specify the gender of a person.
    • Birth date and times should be expressed exactly as indicated in civil registries.
  3. Connected to other data sets.
    Persons should be related to at least one standard id. This is currently not done by the program, but preparatory code has been written to link the data to wikidata.org.
  4. Verified and verifiable.
    An image or a link to the original document.

Trust level

Trust level is an indicator to measure data reliability.
It defines 4 levels of reliability:
  • 1 - Hospital Certificate (HC)
    Original document available and verifiable by anyone.
  • 2 - Birth Certificate (BC)
    Original document available and verifiable by anyone.
  • 3 - Birth Record (BR)
    (= copy of the BC by an officer)
    Original document available and verifiable by anyone.
  • 4 - To Check
    Data possibly containing errors: Gauquelin, Müller, skeptic groups, wikipedia and other web pages.
    Need to be matched against a BC.
Most births that occured in the 19th and early 20th century do not have a HC Hospital Certificate, level 1.
So the best we can hope for OGDB is to reach level 2 Birth Certificate.
Currently, almost all data of OGDB are level 4 To Check.

Remark : Françoise and Michel Gauquelin, and Arno Müller were apparently very serious and were taking a lot of care to avoid errors. But in practice, they never had access to BCs: they were writing to the town halls ; an officer was copying the information contained in the BC and sending them back a letter. They were only recieving BRs.
This is a first source of error, the transcription from BC to BR.
Other sources of errors exist:
  • Gauquelin or Müller error when integrating the BR in their files.
  • Error when original paper files where put in an electronic form.
  • Bugs in g5 program
But today, birth certificates are available online, so we can check and build reliable data. Checking thousands of birth date is a big task, but it's really easier than at Gauquelin epoch.
