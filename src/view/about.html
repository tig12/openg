What is good data ?

The definition of "good data" used in this program is :
  1. Expressed in open and standard formats.
    • Encoded in UTF-8
      The program does not handle other character encodings. Raw data are converted to UTF-8, generated data are UTF-8.
    • Use only open formats
      The program uses txt, csv, json and yaml.
    • Express dates in a standard format
      The program uses ISO 8601 (YYYY-MM-DD HH:MM).
      This choice is not completely satisfactory because ISO 8601 is not fully available for free.
    • Express countries in a standard format
      The program uses ISO 3166 2-letter codes.
    • Relate places to a standard
      The program uses geonames.org.
    • Use a standard list of occupation codes
      among all data. A convenient standard list was not found. The program uses a home-made list of codes
      Most of g5 are occupations are related to Wikidata, but Wikidata was not used as reference list because some codes are missing (eg "Conductor of military band").
  2. Precise and identifiable without ambiguity.
    • Different forms of a person's name are sometimes needed :
      • Exact spelling of family and given names used in birth certificates.
      • Name commonly used to designate the person.
      • Spouse name for women.
      • Nickname or stage name.
    • Specify the gender of a person.
    • Birth date and times should be expressed exactly as indicated in civil registries.
    The structure of an ogdb person is described by github.com/tig12/gauquelin5/blob/master/src/model/Person.yml.
  3. Verified and verifiable.
    Original document (birth certificate) should be available to the user.
    Under development, not finished
  4. Connected to other data sets.
    Persons should be related to at least one standard id.
    Currently not done by the program
    but see preparatory study on tig12.github.io/gauquelin5/wikidata.html.
